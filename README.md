## The Problem

Campus lost and found is broken. Items sit unclaimed in cardboard boxes. Recovery attempts drown in chaotic WhatsApp threads. A "MacBook" report never connects with the "Apple laptop" that was turned in вҖ” because no one's reading every message.

**RETRIVA fixes this.** A centralized, AI-powered platform that understands what you mean вҖ” not just what you type.

<br/>

## What Makes RETRIVA Different

### рҹ§  Multimodal AI at the Core

| Feature | What it does |
|---|---|
| **Auto-Description** | Snap a photo вҶ’ AI extracts brand, color, type, and condition. No manual typing. |
| **Semantic Matching** | Vector search understands that *"MacBook"* = *"Apple laptop"*. Keyword matching can't do this. |
| **Match Comparator** | Side-by-side AI analysis of two items produces a **Match Confidence Score** вҖ” so users verify ownership with evidence, not gut feeling. |

### рҹӣЎпёҸ Guardian AI вҖ” Privacy by Default

Privacy isn't an afterthought. Before any image goes live:

- **Face Detection** вҖ” Uploads containing faces are automatically rejected outright.
- **Document Redaction** вҖ” Student IDs, credit cards, and sensitive text are detected and masked.
- **Content Moderation** вҖ” Spam, pranks, and inappropriate uploads are banned and rejected before they ever appear.

### вҡЎ Real-Time Everything

- **Instant Match Alerts** вҖ” Get notified the moment a potential match is found.
- **Secure In-App Messaging** вҖ” Coordinate retrieval without sharing your phone number.
- **Live Status Tracking** вҖ” Every report is `Open` or `Resolved`. The database stays clean automatically.

## Getting Started

### Prerequisites

- Node.js `v18+`
- A [Firebase project](https://console.firebase.google.com/)
- [Gemini API key](https://aistudio.google.com/) (Google AI Studio)
- A [Cloudinary account](https://cloudinary.com/)

### Setup

**1. Clone the repo**
```bash
git clone https://github.com/your-username/retriva.git
cd retriva
```

**2. Install dependencies**
```bash
npm install
```

**3. Configure environment variables**

Create a `.env` file in the project root:
```env
VITE_API_KEY=your_gemini_api_key
```

> Firebase and Cloudinary configs live in `src/services/firebase.ts` and `src/services/cloudinary.ts`.

**4. Start the dev server**
```bash
npm start
```
